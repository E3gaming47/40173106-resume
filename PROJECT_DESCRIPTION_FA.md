# توضیح پروژه - رزومه آنلاین تعاملی

## هدف پروژه

هدف این پروژه، طراحی یک رزومه آنلاین تعاملی با استفاده از HTML و CSS برای بخش فرانت‌اند و FastAPI برای بخش بک‌اند است. پروژه شامل فرم درخواست پروژه، صفحه مدیریت درخواست‌ها با احراز هویت، و نمایش تعداد کاربران آنلاین به صورت زنده با وب‌سوکت است.

## ویژگی‌های پیاده‌سازی شده

### 1. بک‌اند با FastAPI ✅

#### فرم درخواست پروژه
- کاربران می‌توانند با پر کردن فرم اطلاعات، درخواست پروژه را ثبت کنند
- اطلاعات شامل:
  - نام (Name) - اجباری
  - ایمیل (Email) - اجباری با اعتبارسنجی
  - توضیح کوتاه پروژه (Project Description) - اجباری
  - نوع پروژه (Project Type) - اختیاری
  - بودجه (Budget) - اختیاری
  - زمان‌بندی (Timeline) - اختیاری

#### صفحه ادمین
- نمایش همه درخواست‌های ارسال شده در یک جدول
- احراز هویت با JWT برای دسترسی به صفحه ادمین
- امکان مدیریت درخواست‌ها:
  - مشاهده جزئیات هر درخواست
  - حذف درخواست
  - تغییر وضعیت درخواست (pending, in_progress, completed, rejected)

#### نمایش تعداد کاربران آنلاین
- با استفاده از وب‌سوکت تعداد کاربران آنلاین به صورت زنده در صفحه رزومه نمایش داده می‌شود
- هر بار که کاربری وارد صفحه می‌شود یا از صفحه خارج می‌شود، تعداد آنلاین‌ها بروزرسانی می‌شود
- اتصال خودکار در صورت قطع شدن

### 2. فرانت‌اند

#### صفحه اصلی (index.html)
- نمایش رزومه
- فرم درخواست پروژه
- نمایش تعداد کاربران آنلاین با آیکون و شمارنده زنده

#### صفحه پورتفولیو (portfolio.html)
- نمایش آثار و پروژه‌های انجام شده

#### صفحه ادمین (admin.html)
- فرم ورود با احراز هویت
- جدول نمایش درخواست‌ها
- امکان مدیریت درخواست‌ها

### 3. دیپلوی بک‌اند

بک‌اند پروژه آماده دیپلوی روی سرورهای مختلف است:

#### گزینه‌های دیپلوی:
- **لیارا (Liara)**: سرویس PaaS رایگان ایرانی
- **VPS**: هر سرور مجازی با Python
- **Heroku**: سرویس PaaS بین‌المللی
- **Railway**: سرویس PaaS مدرن

#### لینک OpenAPI (Swagger)
پس از دیپلوی، مستندات API در آدرس زیر در دسترس است:
```
https://<your-server-domain>/docs
```

مثال:
- `https://your-app.liara.run/docs`
- `https://api.yourdomain.com/docs`

## API Endpoints

### احراز هویت
- `POST /api/auth/login` - ورود به سیستم و دریافت JWT token

### درخواست‌های پروژه
- `POST /api/project-requests` - ایجاد درخواست جدید (عمومی)
- `GET /api/project-requests` - دریافت همه درخواست‌ها (نیاز به احراز هویت)
- `GET /api/project-requests/{id}` - دریافت یک درخواست خاص (نیاز به احراز هویت)
- `PATCH /api/project-requests/{id}` - به‌روزرسانی وضعیت درخواست (نیاز به احراز هویت)
- `DELETE /api/project-requests/{id}` - حذف درخواست (نیاز به احراز هویت)

### WebSocket
- `WS /ws/online-users` - اتصال WebSocket برای نمایش تعداد کاربران آنلاین

## نکات مهم برای دیپلوی

1. **تنظیم CORS:** در فایل `backend/main.py`، دامنه فرانت‌اند خود را در تنظیمات CORS قرار دهید.

2. **تغییر URL API:** در فایل‌های HTML، متغیر `API_BASE_URL` را به آدرس سرور خود تغییر دهید.

3. **امنیت:**
   - تغییر رمز عبور ادمین
   - استفاده از HTTPS
   - تنظیم SECRET_KEY به صورت امن
   - محدود کردن CORS

4. **دیتابیس:** در حال حاضر از SQLite استفاده می‌شود. برای production می‌توانید از PostgreSQL استفاده کنید.
